～コード解説～

fn

関数定義の合図

functionの略

main

プログラムのエントリーポイント
必須アイテム

boxの名前
これを忘れるとコンパイルエラーになる！

println!

標準出力（ターミナル等）に文字列を表示させるマクロ

mainの処理（中身）

use

外部のクレートやライブラリなどを呼び出すもの

呼び出せるようにスタンバイさせる
クレート内のものを省略して入力できるようになる

let

変数定義するもの

デフォルトは不変である
let x = 5 と入力する

mut

変数を可変（変更可能）にする

mutable（可変）の略
let mut x = 5と入力する

loop

スコープ内の処理を永久に繰り返す

breakを使用して抜け出せる

break

ループ処理を抜け出すためのもの

例: loop, for, whileで使用可能

continue

今のループ等をスキップして次に行く

例: loop, for, whileで使用可能

String

可変の文字列型

ヒープメモリ（領域）に保存される

io::stdin()

io: input/output
stdin(): 標準入力を受け取る入口

標準入力: キーボードからの入力のこと

.expect()

成功or失敗の型に対して使用できるメゾット

Result型とpanic!を内部で呼び出している
簡単に言えばエラーログ

Result

Rustでは失敗の可能性があるものはこの型で返される

T: 成功したときの型が入っている(指定した型名)
E: 失敗したときのエラーが入っている(e)

panic!

プログラムを強制終了させるマクロ

ifなどで起きてはならない状況を止めれる
ファイル破損の恐れがあるので使用注意！

.trim()

文字列の前後の空白・改行をなくすメゾット

String, str型に有効

.parse()

文字列をほかの型に変形させるメゾット

"123"を整数123に変形させる
エラーハンドリング必須

豆知識！

メゾット＆パス演算子のあとには必ず()がつく！

パス演算子(::)

match

値に応じて分岐する構文

result型に応じて分岐する
Ok(T): OKの時の処理
Err(e): エラーの時の処理

Ok(), Err()

エラーハンドリングの分岐処理

Ok() =>: ()内には成功時に格納したい変数を入れる
Err(e) =>: ()内にはエラー内容を格納するもの(通常はe)

if

条件によって処理を分ける制御構文

条件式はbool型

else if

追加の条件分岐を作る構文

if, ifとするとすべての分岐を試してしまう。
else ifだと上の分岐に当てはまればその他はスキップされる

else

if, else ifに当てはまらなかった条件すべてを処理する構文

安全のために置いておく受け皿

比較演算子

!=

<

<=
